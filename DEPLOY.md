# Vesper Obsidian Nexus 部署与使用手册

本手册假设操作者完全没有经验、只拥有一台 Windows 设备进行首轮部署，之后可以在任何移动端或桌面端直接访问同一份代码。流程分为准备环境、启动预览、常见操作以及常见问题排查。每一步都不要跳过。

## 1. 准备环境（Windows 首次部署）
1. 打开浏览器访问 [https://nodejs.org](https://nodejs.org) 并下载安装 **LTS 版本**的 Node.js，安装时保持默认选项并勾选“自动添加到 PATH”。
2. 安装完成后，按 `Win + R` 输入 `cmd` 回车，执行 `node -v` 与 `npm -v`，若能输出版本号则环境就绪。
3. 在桌面或任意文件夹空白处右键选择“新建”→“文件夹”，命名为 `vesper-nexus`，用于存放项目。
4. 从代码仓库下载最新压缩包或通过 Git 克隆：
   - 下载压缩包：将 ZIP 解压到 `vesper-nexus` 文件夹。
   - 使用 Git：在 `cmd` 中执行 `cd 路径\vesper-nexus && git clone <仓库地址> .`。
5. 双击进入 `vesper-nexus` 目录，确认其中包含 `目前很少的代码.html`、`README.md`、`DEPLOY.md`、`FILES.md` 等文件。

## 2. 本地预览与二次访问
1. 在 `cmd` 中切换到项目目录：`cd 路径\vesper-nexus`。
2. 直接运行：`start 目前很少的代码.html`（Windows）或双击该文件，即会在默认浏览器打开页面。
3. 若需要通过本地服务器以便移动端访问，可执行：
   ```bash
   npx serve .
   ```
   终端会显示一个本地地址（如 `http://localhost:3000`），确保防火墙允许本地访问，然后在同一局域网内用手机输入该地址即可。
4. 之后的所有访问都可以在手机或桌面浏览器直接打开同一文件或同一局域网地址，无需再次安装 Node.js。

## 3. 页面主要操作指南
- **主题与个性化**：点击侧栏的「偏好与主题」按钮，按提示输入 `graphite`、`dusk` 或 `quartz` 即可切换，也可直接粘贴自定义 CSS 代码。
- **搜索**：顶部搜索框可实时过滤当前分类的图标；上方 Chips 可快速切换关注的分区（图标/网盘/资源/笔记/博客/AI/工具）。
- **图标管理（开发者）**：
  1. 点击「开发者模式」输入识别码，初始最高权限码为 `Vesper`，首次输入会自动与当前设备绑定。
  2. 进入开发者后可拖拽图标排序，三击删除，点击「新增图标/链接」可增加新分组或新条目。
  3. 右键或单击图标打开菜单，可复制链接/代码或转为 ICO/下载。
- **账户与登录**：
  1. 在「账户与权限」面板填写邮箱、昵称、密码，点击「发送验证码」后记录弹出的验证码，再输入后完成注册。
  2. 已注册用户可登录、退出，若忘记密码可点击“找回密码”重新设置。
- **VIP 与优惠码**：登录后点击「开通 VIP」输入套餐（“月卡/季卡/年卡”）与优惠码。优惠码由最高权限生成，支持折扣与免费体验。
- **开发者识别码管理（最高权限）**：使用 `Vesper` 或其他最高权限码进入后，打开「最高管理者控制台」：
  - 「开发者识别码」可批量生成、查看绑定状态、收回。
  - 「优惠码」可生成 9 折、7 折、5 折、3 折、免费月/周/季/年卡或永久码，并可收回。
  - 「用户总览」可封禁账号、授予 VIP、生成新的开发者识别码指派给用户。

## 4. 移动端与自适应
- 页面采用响应式布局，窄屏下侧栏可折叠，卡片网格自动调整。移动端访问同一地址即可。

## 5. 常见问题与排查
- **页面空白或样式异常**：清理浏览器缓存或使用无痕模式重新打开；确认 `目前很少的代码.html` 没被移动或重命名。
- **复制失败**：部分浏览器限制剪贴板，需要在安全环境（https 或本地文件）下触发，并确保未禁止剪贴板权限。
- **图片无法加载**：检查外链是否被网络阻断，可替换为可访问的 CDN 链接后再使用「新增图标」更新。
- **开发者识别码无效**：确认是否输入正确大小写；若提示已绑定，请在原设备解除或让最高权限收回后重新生成。
- **本地存储丢失**：数据存储在浏览器 localStorage 中，清空缓存会重置；可考虑自行备份 `localStorage` 导出或将同名文件放到简易服务器以保持路径稳定。

## 6. 安全提示
- 本项目前端无后端依赖，所有数据仅保存在本地浏览器存储，用于演示。若需生产环境，请补充后端鉴权、数据库与支付回调校验逻辑。
- 不要在公共电脑输入真实支付或隐私信息，示例的支付/优惠逻辑均为演练模型。

完成以上步骤即可获得一个可在 Windows 首次部署、随后任意设备访问的冷淡风格多功能工作台。祝使用愉快。
